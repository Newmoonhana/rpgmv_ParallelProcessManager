RPG MV의 병렬 이벤트의 세부 관리 및 최적화를 지원하는 플러그인<br>

 * 주요 개선 사항<br>
   알만툴 시리즈에는 병렬 이벤트를 여러개 만들 시 or (이벤트 한개더라도) 너무 많은 이벤트 코드를 처리할 시 게임이 렉이 걸리는 현상이 있습니다.<br>
   최대한 간략하게 설명하자면 1프레임 내로 너무 많은 작업을 하려고 해서 일어나는 프레임 멈춤 현상입니다.(좀 다르긴함. 이해 측면에서 설명한거)<br>
   이는 사실 유니티같은 다른 프로그램에서도 존재할 수 있으나, 알만툴 특유의 이벤트 페이지 내 보이지 않는 대기 프레임으로 인해 알만툴의 큰 단점처럼 느껴집니다.<br>
   1차적으로는 이 문제를 완화시키고자 해당 플러그인을 개발했습니다. 물론 특정 프레임에서 너무 많은 이벤트가 호출된다면 렉 걸리는 문제 자체가 해결된 건 아니지만, 사용자의 역량에 따라 더 완화시킬 수 있습니다.<br>
   2차적으로는 병렬 이벤트 내의 불필요하게 자추 호출되는 코드들을 덜 호출되도록 합니다. 굳이 1프레임마다 호출할 필요가 없는 이벤트는 조건 분기로 특정 프레임에만 호출될 수 있도록 합니다.<br>
   사실 이건 유니티같은 다른 에디터에선 해당 코드에서마다 수동으로 기법을 넣는게 일반적인데, 블록 코딩인 알만툴에서는 그렇게 넣으면 지리멸절해지기 쉽기 때문에... 플러그인 도움을 받으면 좋을겁니다.<br>
   ( 해당 최적화 기법에 대해 설명하는 영상: https://youtube.com/shorts/BuvVXUpVgfI?si=nbtvSH_-41Wl5AAa )<br>
   3차적으로는 결과적으로 병렬 이벤트 내에서 조건분기로 코드 영역을 확실히 분리시켜주는 느낌이 있어, 스파게티 코드를 방지할 수 있습니다. 다만 여기부터는 사용자 실력에 따라 편차가 좀 갈릴 것으로 보입니다.<br>
